# Prototype: Echoes of the Tavern (Text RPG)

Прототип текстовой RPG с упором на математически выверенную прогрессию и баланс. Игра сочетает жесткую логику .NET бэкенда и нарративную генерацию через LLM (Qwen).

## 1. Концепция
*   **Жанр:** Текстовое приключение / Dungeon Crawler.
*   **Сеттинг:** Реалистичное средневековье.
*   **Цель:** Пройти цепочку локаций за 10 минут, добравшись до финала через бои или дипломатию.
*   **Особенность:** Игрок должен запоминать детали сюжета (имена, пароли, события), чтобы избегать лишних сражений.

## 2. Игровой цикл (Core Loop)
1.  **Narrative Phase:** Получение описания ситуации (генерация Qwen).
2.  **Decision Phase:** Выбор действия (кнопки/варианты).
3.  **Resolution Phase:** Расчет результата на сервере (бросок кубика + модификаторы статов).
4.  **Progression Phase:** Получение опыта/лута и переход к следующему узлу сюжета.

## 3. Математическая модель и Баланс
Для обеспечения 10-минутной сессии прогрессия рассчитана на **5-7 ключевых сцен**:

*   **Характеристики:**
    *   `HP` (Здоровье): Базовое 100.
    *   `Level` (Уровень): 1-5 (быстрая кривая опыта).
    *   `Armor` (Броня): Прямое снижение входящего урона (Flat Reduction).
*   **Боевой баланс:**
    *   Вероятность попадания: `Base (60%) + (Level_diff * 5%)`.
    *   Урон оружия: `Base_Damage + (Level * Modifier)`.
*   **Экономика:** Золото тратится на расходники, повышающие шансы в финальной сцене.

## 4. Прогрессия и Сюжетные узлы
1.  **Таверна (Старт):** Выбор снаряжения, получение первой зацепки.
2.  **Дорога (Минорные квесты):** 2-3 случайных события для гринда опыта (проверка баланса статов).
3.  **Сюжетный барьер:** Проверка памяти игрока. Если игрок не знает "код/пароль", вступает в тяжелый бой.
4.  **Финал:** Босс локации. Баланс рассчитан так, что без снаряжения из п.2 шанс на победу < 20%.

## 5. Технологический стек
*   **Backend:** .NET 8 (Core logic, Balance Engine).
*   **Database:** SQLite/PostgreSQL (Лидерборд, сессии).
*   **Frontend:** Чистый JS + CSS (Retro terminal style).
*   **AI:** Qwen 2.5b (Локальный запуск) — генерирует описания по шаблонам от сервера.
